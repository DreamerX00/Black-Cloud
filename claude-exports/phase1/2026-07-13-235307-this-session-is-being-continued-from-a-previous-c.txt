  ★ Insight ─────────────────────────────────────
  The general rule for SSR-safe reduced-motion: never branch DOM output on a 
  hook that reads the browser. Two safe alternatives:
  1. CSS-first gating — motion-reduce:animate-none, motion-reduce:transform-none
  Tailwind variants translate to @media (prefers-reduced-motion: reduce) at CSS
  level, so the DOM is identical and only styles change. Cost: zero JS, no
  hydration risk.

  transitions instead of you branching props. The initial rendered frame is
  still the same on both sides; only the animation between frames is neutered.

  useReducedMotion() when you need the raw boolean for an imperative side-effect
  (like HeroScene gating a dynamic import), and even then read it via
  useSyncExternalStore with a stable getServerSnapshot.
