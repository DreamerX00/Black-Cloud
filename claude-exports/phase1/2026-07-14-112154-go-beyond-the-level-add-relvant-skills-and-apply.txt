  the underlying primitive lowers to element.animate(), the browser applies the
  stricter Web Animations rule. This is why the crash only surfaces at runtime вҖ”

  clamping to [0,1] isn't enough for the last item: clamped bounds could
  collapse to [1, 1], and equal offsets on a two-stop keyframe list also throw
  in some browsers.
  - The other useTransform calls (aurora, chapter, horizontal-gallery) all pass
  literal, hand-picked keyframes so they never drift into this failure mode.
  Rule of thumb going forward: any time a keyframe array is computed from
  indices/props, clamp both ends.
