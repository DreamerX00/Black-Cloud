  What was wrong

  useState(() => hasWebGL()) ran its initializer during the client's first 
  render, where window exists вҖ” so it returned true on the client but false on
  the server (no WebGL in SSR). The server sent the StaticFallback markup; the

  hydration error, whole subtree regenerated.

  The fix

  Detect WebGL in a mount effect instead:

  useEffect(() => setWebgl(hasWebGL()), []);
  Effects never run during SSR or the first hydration render, so both sides now
